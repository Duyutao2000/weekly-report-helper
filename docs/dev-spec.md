# 开发文档

## 1. 项目概述

个人周报助手，本地待办事项管理工具，支持每周任务追踪与一键生成周报。

## 2. 核心功能

### 2.1 任务管理

| 功能 | 说明 |
|------|------|
| 新增任务 | 选择项目、填写任务描述、填写工时 |
| 完成任务 | 标记任务为已完成 |
| 查看任务 | 按项目分组展示，显示工时分配情况 |

### 2.2 周期管理

- 支持自定义日期范围（手动选择起止日期）
- 一周默认为 5 个工作日，总工时 40 小时
- 切换周期时清空当周任务视图

### 2.3 工时管理

| 规则 | 说明 |
|------|------|
| 总工时 | 40 小时/周（5 工作日 × 8 小时） |
| 分配方式 | 每个任务手动填写工时 |
| 剩余计算 | 自动计算未分配工时：`剩余 = 40 - 已分配总和` |
| 校验 | 提交时检查工时是否为 0，提示用户 |

### 2.4 下周计划

| 功能 | 说明 |
|------|------|
| 新增计划任务 | 选择项目、填写任务描述（无工时） |
| 生成计划 | 调用大模型润色输出 |
| 同步到下周 | 将上周计划同步为本周待办任务 |

**输出格式**：
```
[项目名称]

1. [任务描述1]
2. [任务描述2]
```

**同步逻辑**：
1. 切换周期时，检测上周是否有未同步的计划任务
2. 弹窗提示：发现 N 条上周计划，是否同步为本周任务？
3. 确认后，将上周计划任务复制到本周任务列表（状态为 pending，工时为空待填写）

### 2.5 周报生成

**输入**：当周已完成的任务列表（含项目、任务描述、工时）

**输出格式**：
```
[项目名称] 工时：XX小时

1. [任务描述1]
2. [任务描述2]

[下一个项目...]
```

**处理逻辑**：
1. 从数据库读取当周已完成任务
2. 按项目分组，汇总每个项目的工时
3. 调用大模型 API，prompt 包含：
   - 原始任务列表
   - 输出格式要求
   - 书面化润色要求
4. 返回生成的周报文本

## 3. 数据模型

### 3.1 项目表 (projects)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| name | VARCHAR(100) | 项目名称 |
| created_at | DATETIME | 创建时间 |

### 3.2 任务表 (tasks)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| project_id | INTEGER | 关联项目 |
| description | TEXT | 任务描述 |
| hours | FLOAT | 工时（小时） |
| status | VARCHAR(20) | pending/completed |
| week_start | DATE | 周开始日期 |
| week_end | DATE | 周结束日期 |
| created_at | DATETIME | 创建时间 |
| completed_at | DATETIME | 完成时间 |

### 3.3 下周计划表 (next_week_plans)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| project_id | INTEGER | 关联项目 |
| description | TEXT | 任务描述 |
| week_start | DATE | 所属周开始日期 |
| synced | BOOLEAN | 是否已同步到下周（默认 false） |
| created_at | DATETIME | 创建时间 |

### 3.4 周报表 (reports)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| week_start | DATE | 周开始日期 |
| week_end | DATE | 周结束日期 |
| content | TEXT | 生成的周报内容 |
| created_at | DATETIME | 生成时间 |

## 5. 技术实现

### 5.1 项目结构

```
weekly-report-helper/
├── app.py              # Gradio 主界面
├── models.py           # SQLAlchemy 数据模型
├── services/
│   ├── task.py         # 任务 CRUD
│   ├── plan.py         # 下周计划 CRUD + 同步
│   ├── report.py       # 周报生成
│   └── llm.py          # 大模型 API 调用
├── templates/
│   ├── report_prompt.txt   # 周报生成 prompt
│   └── plan_prompt.txt     # 计划生成 prompt
├── .env                # API Key 配置
├── requirements.txt    # 依赖
└── data/
    └── todo.db         # SQLite 数据库
```

### 5.2 依赖

```
gradio>=4.0
sqlalchemy>=2.0
openai>=1.0
python-dotenv>=1.0
```

### 5.3 大模型配置

```env
# .env
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen-plus
```
